# CAS Attendance Core

> نسخه: `19.0.1.0.2` · Odoo 19 Community

ثبت ممیزی‌پذیر رخداد دستگاه/نگهبانی و تطبیق آنها با برنامه شیفت در سطح روز.

## اجزای ماژول

- `cas.attendance.site` و device: محل و دستگاه
- `cas.attendance.outage`: خرابی دستگاه
- `cas.attendance.event`: رخداد خام append-only
- `cas.attendance.day`: تطبیق روزانه
- `cas.attendance.interval`: بازه‌های حالت پیشرفته

## نقش‌ها

کاربر، نگهبان ثبت‌کننده، سرپرست، مدیر و دریافت‌کننده دستگاه.

## روش کار

1. رخداد دستگاه یا نگهبانی با منبع و زمان ثبت می‌شود.
2. رخداد به روز کاری و شیفت منتشرشده نسبت داده می‌شود.
3. روز حضور منابع را مقایسه و ورود/خروج مؤثر را محاسبه می‌کند.
4. تعارض برای سرپرست باز می‌شود تا منبع یا مقدار سفارشی را با دلیل انتخاب کند.
5. ابطال رخداد با حفظ اصل و در صورت نیاز رخداد جایگزین انجام می‌شود.

## منوها

حضور من، ورود/خروج من، ثبت و رسیدگی حضور، خرابی‌ها، محل‌ها و دستگاه‌ها.

## نصب و ارتقا

وابستگی‌ها: `cas_shift_management`، `mail`.

```bash
./odoo-bin -d <database> -i cas_attendance_core --stop-after-init
./odoo-bin -d <database> -u cas_attendance_core --stop-after-init
```

## قواعد کلیدی

- رخداد خام پس از ثبت ویرایش/حذف عادی نمی‌شود.
- تصمیم تعارض باید مسئول، زمان و دلیل داشته باشد.
- دسترسی کاربر عادی فقط سوابق خود اوست.

## آزمون‌ها

آزمون‌های attribution، reconciliation، conflict، void و security.
