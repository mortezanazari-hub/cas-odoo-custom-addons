# ارزیابی اثر نسخه ۷ بر `cas_action_hub`

| مشخصه | مقدار |
|---|---|
| وضعیت | `Needs Review` |
| نوع سند | ارزیابی اثر؛ نه مجوز اجرا |

## تغییر اصلی

در نسخه ۴ Route `my-actions` با عنوان «کارهای من» نمایش داده می‌شد. در نسخه ۷ این Route به «نیازمند اقدام» تغییر معنا داده و کارهای شخصی به Route و Capability مستقل منتقل شده‌اند.

## آثار

- جداسازی Action رسمی از Personal Task
- Provider Widget «نیازمند اقدام» در میزکار
- ارائه Deadline برای Calendar Aggregator
- ارائه Notification برای اقدام جدید یا نزدیک موعد
- ارائه Search Provider
- Deep Link به رکورد منبع
- استفاده از واژگان کاربرپسند به‌جای SLA در UI کاربر عادی

## قواعد

- Action Hub مالک منطق تخصصی رکورد منبع نیست.
- Action فقط Resource Reference، مسئول، اولویت، Deadline، SLA داخلی و Route را نگه می‌دارد.
- تکمیل Action باید پس از اجرای متد دامنه ماژول منبع همگام شود.
- Personal Task نباید بدون تصمیم جداگانه وارد این ماژول شود.

## سؤال‌های باز

- منبع واحد برنامه امروز چیست؟
- آیا Reminderهای Action از Notification Service مشترک استفاده می‌کنند؟
- سیاست نمایش Actionهای چندنقشی چگونه است؟
