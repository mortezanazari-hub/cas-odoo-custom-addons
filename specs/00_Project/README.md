## قاعده استفاده

هر سند جدید باید:

1. نسخه محصول و وضعیت خود را مشخص کند.
2. به Decisionها، صفحه‌ها، ماژول‌ها و Architecture Contractهای مرتبط لینک دهد.
3. در صورت جایگزینی سند قبلی، رابطه `supersedes` را صریح ثبت کند.
4. برای اجرا، API، Security، Migration و Test Strategy قابل‌ردیابی داشته باشد.
