# تاریخچه نسخه‌های رسمی رابط CAS

| مشخصه | مقدار |
|---|---|
| وضعیت | `Agreed` |
| دامنه | Product / UI Versioning |

## خط مبنا

آخرین خط مبنای رسمی پیش از بازنگری گسترده Workspace، بسته `CAS_UI_Prototype_V4` است.

نسخه‌های ۵ و ۶ به‌عنوان Release رسمی مستقل ثبت نمی‌شوند و صرفاً Iterationهای داخلی طراحی بوده‌اند.

```text
CAS UI Prototype v4 → CAS UI Workspace v7
```

## نسخه ۴

- معماری نقش‌محور
- صفحات تخصصی ماژول‌ها
- نگهبانی کارت‌محور
- داشبوردهای سرپرست، مدیر و مدیرعامل
- قراردادهای اولیه Workspace و Odoo

## نسخه ۷

- توسعه میزکار کاربر عادی به مرکز فرمان شخصی
- سیستم Widget قابل جابه‌جایی
- تقویم تعاملی داخل میزکار
- گفتگو به‌عنوان قابلیت سطح اول
- Theme و خوانایی سراسری
- Sidebar جمع‌شونده
- ارتفاع ثابت Widgetها و Scroll داخلی

## قاعده نسخه‌بندی آینده

یک نسخه فقط زمانی Release رسمی محسوب می‌شود که:

1. خط مبنا و دامنه آن در این سند ثبت شود.
2. اسناد صفحه در `02_UI_UX` به‌روزرسانی شوند.
3. تصمیم‌های مشترک در `04_Decisions` ثبت شوند.
4. Change Set و ماتریس تجمیع به‌روزرسانی شوند.
5. در صورت آماده‌بودن پیاده‌سازی، Specification ماژولی تولید شود.
