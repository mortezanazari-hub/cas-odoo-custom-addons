# تصمیم‌های محصول Workspace در نسخه ۷

## خط مبنا

این تصمیم‌ها بر پایه `CAS_UI_Prototype_V4` ثبت شده‌اند. نسخه‌های ۵ و ۶ Release رسمی مستقل محسوب نمی‌شوند.

## تصمیم‌های قطعی

### ۱. Workspace جایگزین Odoo Core نیست
تمام پیاده‌سازی در Custom Addons انجام می‌شود و تغییر Core ممنوع است.

### ۲. Workspace مالک داده کسب‌وکاری نیست
Workspace فقط مسئول Shell، Navigation، Dashboard، Widget، Drawer، Route Resolution و Action Dispatch است.

### ۳. میزکار Action-First است
اولویت نمایش با کارهای کاربر، تصمیم‌های نیازمند اقدام، جلسات نزدیک، گفتگوهای خوانده‌نشده و گزارش فعالیت/حضور است.

### ۴. Hero کوتاه و ساختاریافته است
- ردیف خوش‌آمدگویی و وضعیت حضور
- چهار کارت وضعیت هم‌اندازه
- جست‌وجوی سراسری تمام‌عرض

### ۵. Widgetها قابل شخصی‌سازی هستند
در Prototype ترتیب در Local Storage و در Production در تنظیمات کاربر Odoo ذخیره می‌شود.

### ۶. ارتفاع Widgetها ثابت است
- Header ثابت
- Footer ثابت
- Body قابل Scroll
- فهرست کم‌داده با ردیف خالی غیرفعال تکمیل می‌شود

### ۷. گفتگو یک قابلیت سطح اول است
- عنوان نمایشی: «گفت‌وگوها»
- Route فنی: `messages`
- دسترسی از Sidebar، Topbar و میزکار
- آیکن دو حباب گفتگو
- اطلاعات جانبی در Drawer

### ۸. تقویم میزکار تعاملی است
نمای روز، هفته و ماه بدون ترک میزکار تغییر می‌کند. نمای ماهانه باید ماه، سال، روزهای هفته، روز جاری و روزهای دارای رویداد را نشان دهد.

### ۹. شخصی‌سازی ظاهری کنترل‌شده است
- اندازه متن
- تراکم رابط
- Accent Color
- Light/Dark Mode
- وضعیت Sidebar

### ۱۰. خوانایی بر تراکم اولویت دارد
هیچ متن کاربردی کمتر از ۱۲ پیکسل نیست. حالت فشرده فقط فاصله‌ها را کاهش می‌دهد.

### ۱۱. Sidebar واقعاً Collapse می‌شود
- کنترل بازکردن همیشه در دسترس
- آیکن‌ها باقی می‌مانند
- Labelها مخفی می‌شوند
- Tooltip در نظر گرفته می‌شود
- وضعیت کاربر حفظ می‌شود

### ۱۲. UI مرجع امنیت نیست
مرجع نهایی امنیت ACL، Record Rule، Method-level Check و Domain Scope است.

## موارد باز برای پیاده‌سازی

- مدل ذخیره تنظیمات شخصی Workspace
- Widget Registry
- Server-side Drag Order
- اتصال Mail/Bus
- اتصال Calendar واقعی
- Notification Service
- سیاست Theme سازمانی
- قرارداد Cache و Offline State
