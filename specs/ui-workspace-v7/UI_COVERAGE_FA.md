# ماتریس پوشش رابط کاربری CAS

## اصل طراحی

- Workspace فقط پوسته، ناوبری، صفحه و Drawer را مدیریت می‌کند.
- Action Hub فقط metadata اقدام و لینک رکورد منبع را نگه می‌دارد.
- Form Builder و Workflow Designer فقط نسخه پیش‌نویس هسته خود را ویرایش می‌کنند.
- Approval مالک تصمیم رسمی است؛ Workflow مالک وضعیت و مسیر فرایند باقی می‌ماند.
- Document Core مالک نسخه فایل و پیوند است.
- Jalali فقط لایه ورود و نمایش است.

## زاویه‌های نقش‌محور

### کاربر عادی
خانه شخصی، کارتابل، فرم‌های قابل ثبت، ثبت‌های من، گردش‌کارهای من، مکاتبات، اسناد، شیفت من، حضور من، کاردکس من، مرخصی/مأموریت، اضافه‌کاری و گزارش کار.

### سرپرست
داشبورد تیم، صندوق تأیید، مغایرت‌های حضور، گزارش‌های حوزه، عملیات کاردکس و خروجی کاردکس.

### مدیر واحد
داشبورد مدیریت، سیاست تأیید، SLA، برنامه‌ریزی شیفت، دوره‌های کاردکس، ریسک‌ها و گزارش‌های مدیریتی.

### مدیرعامل
داشبورد تجمیعی، تصمیم‌های سطح عالی، دفتر امضا، وضعیت واحدها و قفل/بازگشایی‌های حساس.

### نگهبان
ثبت ورود و خروج کارت‌محور، جست‌وجوی نام/کد/سمت، فیلتر واحد، تغییر ساعت دستی، محل ثبت، یادداشت و لاگ رخدادها.

### دبیرخانه
دفتر وارده و صادره، صف نیازمند ثبت، قالب‌های رسمی، تولید PDF و دفتر امضا.

### برنامه‌ریز شیفت
انتساب الگو، انتشار Snapshot، ورود فایل دستگاه، Staging و نگاشت شناسه‌های خارجی.

### طراح فرم
مدیریت فرم، نسخه‌ها، Form Builder، پیش‌نمایش و انتشار.

### طراح گردش‌کار
مدیریت فرایند، نسخه‌ها، بوم گراف، Binding فرم، اعتبارسنجی مسیر و انتشار.

### مدیر اسناد
کتابخانه، نسخه‌ها، صف OCR، Backend ذخیره‌سازی، ارائه‌دهنده OCR و Retention.

### مدیر سامانه
سلامت ماژول‌ها، Jalali Center، Workspace Settings، ماتریس نقش/صفحه و وضعیت Cron/Asset/Record Rule.
