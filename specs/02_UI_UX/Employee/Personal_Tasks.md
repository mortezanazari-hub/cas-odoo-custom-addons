# سند تصمیم صفحه کارهای شخصی

| مشخصه | مقدار |
|---|---|
| شناسه | `PAGE-EMP-TASK-001` |
| خط مبنا | `CAS_UI_Prototype_V4` |
| نسخه هدف | `CAS UI Workspace v7` |
| وضعیت محصول | `Agreed` |
| وضعیت معماری | `Needs Review` |
| وضعیت پیاده‌سازی | فعلاً آغاز نشود |
| نقش‌ها | همه کاربران دارای Capability مربوط |

## زمینه تغییر

در نسخه ۴، عنوان «کارهای من» عملاً به Route کارتابل اقدام‌ها (`my-actions`) متصل بود. در نسخه ۷، مفهوم کار شخصی از «اقدام سازمانی» جدا شده و Route مستقل `personal-tasks` با Capability مستقل `personal.tasks` ایجاد شده است. Route قبلی `my-actions` باقی مانده ولی عنوان آن به «نیازمند اقدام» تغییر کرده است.

## هدف

ایجاد فضای مستقل برای کارهای شخصی و روزمره‌ای که الزاماً از Workflow، Approval، Correspondence یا مدل کسب‌وکاری خاصی تولید نشده‌اند.

## تصمیمات قطعی

### `PAGE-EMP-TASK-DEC-001` — جداسازی کار شخصی از Action Hub
`personal-tasks` مالک تجربه کار شخصی است و `my-actions` فقط اقدام‌های سازمانی منتظر انجام یا تصمیم را نمایش می‌دهد.

### `PAGE-EMP-TASK-DEC-002` — Capability مستقل
دسترسی صفحه با Capability `personal.tasks` کنترل می‌شود و به همه نقش‌های عملیاتی و مدیریتی تخصیص داده شده است.

### `PAGE-EMP-TASK-DEC-003` — ثبت سریع
کاربر باید بتواند از میزکار و صفحه کامل، کار جدید را با عنوان کوتاه ثبت کند.

### `PAGE-EMP-TASK-DEC-004` — وضعیت انجام
کار شخصی قابلیت انجام/بازگشایی دارد و تغییر وضعیت باید بدون خروج از صفحه انجام شود.

### `PAGE-EMP-TASK-DEC-005` — زمان‌بندی و انتقال
کار می‌تواند برای امروز، فردا یا تاریخ مشخص برنامه‌ریزی شود. عملیات «انتقال به فردا» باید اقدام مستقیم باشد.

### `PAGE-EMP-TASK-DEC-006` — تبدیل به گزارش فعالیت
در صورت ارتباط واقعی با کار انجام‌شده، کار شخصی می‌تواند نقطه شروع ثبت ردیف گزارش روزانه باشد؛ اما حذف یا تکمیل Task نباید به‌تنهایی گزارش رسمی ایجاد کند.

### `PAGE-EMP-TASK-DEC-007` — نمایش در Widget
میزکار خلاصه‌ای از کارهای باز را نمایش می‌دهد و صفحه کامل مسئول فیلتر، مرور و مدیریت جزئیات است.

## اثر بر ماژول‌ها

| ماژول/دامنه | اثر |
|---|---|
| `cas_workspace` | Route، Widget، صفحه، State و Preference |
| `cas_action_hub` | جداسازی معنایی Action سازمانی از Task شخصی |
| `cas_work_report` | Adapter اختیاری برای تبدیل Task به ردیف فعالیت |
| Notification/Calendar | یادآوری و زمان‌بندی آینده |

## قواعد کسب‌وکار

- Task شخصی مرجع Approval یا Workflow نیست.
- Task شخصی نباید بدون اقدام صریح کاربر، رکورد رسمی گزارش کار بسازد.
- Scope پیش‌فرض Task متعلق به کاربر جاری است.
- دسترسی مدیر یا سرپرست به Task شخصی دیگران پیش‌فرض نیست و نیازمند تصمیم جداگانه است.
- حذف Task باید از تکمیل Task متمایز باشد.

## حالت‌های صفحه

`loading`، `empty`، `ready`، `error` و در صورت غیرفعال بودن سرویس `unavailable`.

## معیارهای پذیرش

- Route مستقل `personal-tasks` در Navigation نمایش داده شود.
- `my-actions` با عنوان «نیازمند اقدام» باقی بماند.
- ثبت سریع، تکمیل، بازگشایی، انتقال به فردا و زمان‌بندی قابل انجام باشد.
- Widget میزکار و صفحه کامل از یک منبع داده واحد استفاده کنند.
- امنیت سمت سرور مستقل از مخفی‌سازی UI باشد.

## سؤال‌های باز

1. Task شخصی در خود `cas_workspace` ذخیره می‌شود یا ماژول مستقل نیاز دارد؟
2. Reminder با Calendar، Notification Service یا Cron پیاده‌سازی می‌شود؟
3. آیا اشتراک Task یا تفویض آن در نسخه اول مجاز است؟
4. سیاست نگهداری Taskهای تکمیل‌شده چیست؟
