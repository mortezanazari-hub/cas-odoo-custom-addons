# DEC-014 — استفاده از قابلیت‌های استاندارد Discuss و قرارداد تعامل پیام

| مشخصه | مقدار |
|---|---|
| وضعیت | `Agreed` |
| نسخه | `CAS UI Workspace v8` |
| دامنه | Conversations / Mail / Discuss / Bus / Message UX |
| صفحات مرتبط | `02_UI_UX/Employee/Conversations.md` |

## زمینه

نسخه ۷ گفت‌وگو را به قابلیت سطح اول Workspace تبدیل کرد. نسخه ۸ رفتار صفحه را به سطح یک پیام‌رسان سازمانی عملیاتی ارتقا می‌دهد، اما نباید مدل پیام‌رسانی موازی و ناقص در `cas_workspace` ساخته شود.

## تصمیم معماری

مالک Conversation، Channel، Message، Member، Attachment، Reaction، Thread و وضعیت خوانده‌نشده باید Odoo Mail/Discuss/Bus یا Extension استاندارد آن باشد. `cas_workspace` فقط Shell، Adapter و تجربه اختصاصی CAS را ارائه می‌کند.

## تغییرات قطعی UI

- نماد گفت‌وگو حباب گفت‌وگو است و از آیکن نامه استفاده نمی‌شود.
- عنوان و توضیح معرفی غیرضروری بالای صفحه حذف می‌شود.
- جست‌وجوی تکراری کنار دکمه گفت‌وگوی جدید حذف می‌شود؛ جست‌وجوی داخل فهرست گفتگو حفظ می‌شود.
- دکمه گفت‌وگوی جدید به‌صورت Floating Action روی فهرست گفتگوها قرار می‌گیرد.
- آواتارها دایره‌ای، هم‌اندازه و دارای Fallback ثابت هستند.
- ردیف‌های فهرست گفتگو فشرده‌اند و Preview حداکثر یک خط است.
- کل صفحه اسکرول نمی‌خورد؛ فهرست گفتگو و بدنه پیام‌ها نواحی Scroll مستقل و کنترل‌شده دارند.
- Composer همیشه در Viewport گفتگو باقی می‌ماند.

## عملیات پیام

Context Menu اختصاصی پیام حداقل شامل این موارد است:

- پاسخ
- فوروارد
- کپی متن
- پین/برداشتن پین
- واکنش Emoji
- مشاهده اطلاعات پیام
- حذف، فقط در صورت مالکیت و سیاست مجاز

عملیات قابل توسعه مانند ویرایش پیام، علامت‌گذاری و لینک مستقیم باید با قابلیت واقعی Mail/Discuss و سیاست سازمان تطبیق داده شوند.

## عملیات گفتگو

Context Menu فهرست گفتگو حداقل شامل:

- پین/برداشتن پین گفتگو
- بی‌صدا/فعال‌کردن اعلان
- آرشیو
- علامت‌گذاری خوانده/خوانده‌نشده
- مشاهده اطلاعات و اعضا
- ترک کانال، در صورت مجازبودن

حذف Conversation فقط مطابق مالکیت و سیاست Odoo مجاز است و نباید به‌عنوان عملیات عمومی نمایش داده شود.

## Reply، Forward و Reaction

- Reply باید Preview پیام مرجع را در Composer و پیام نهایی نشان دهد.
- Forward باید مقصد مجاز را از Channel/Partnerهای قابل دسترس انتخاب کند.
- Reaction باید از مدل استاندارد یا Extension سازگار استفاده کند.
- Emoji Picker در Composer و Reaction Picker باید Keyboard Accessible باشد و با کلیک بیرون بسته شود.

## Context Menu

- منوی مرورگر برای محدوده‌های پیام و گفتگو با منوی اختصاصی جایگزین می‌شود.
- کلیک بیرون، Escape، تغییر گفتگو و Scroll معنادار باید منو را ببندد.
- منو نباید خارج از Viewport رندر شود.
- عملیات غیرمجاز اصلاً نمایش داده نشود یا با دلیل روشن غیرفعال باشد.

## امنیت و مجوز

- مشاهده گفتگو تابع عضویت و Record Rule است.
- حذف پیام تابع نویسنده، بازه زمانی و سیاست سازمان است.
- Forward نباید دسترسی به Attachment یا متن محرمانه را دور بزند.
- Pin، Mute و Archive باید Scope روشن کاربر/کانال داشته باشند.
- Attachment Download علاوه بر عضویت گفتگو، مجوز Attachment/Document را بررسی می‌کند.
- Author پیام از Session معتبر گرفته می‌شود.

## Realtime

Unread Count، Message Delivery، Reaction و Member State باید از Bus یا سازوکار استاندارد Realtime دریافت شوند. Polling فقط Fallback کنترل‌شده است.

## معیار پذیرش

- هیچ اسکرول صفحه‌ای Composer را از دسترس خارج نکند.
- Context Menu با کلیک بیرون و Escape بسته شود.
- Emoji Picker در هر دو محل باز و قابل استفاده باشد.
- ردیف گفتگو تراکم مناسب داشته باشد.
- عملیات Reply، Forward، Pin و Reaction به رکورد واقعی Discuss متصل شوند.
- Workspace هیچ مدل موازی Message ایجاد نکند.
