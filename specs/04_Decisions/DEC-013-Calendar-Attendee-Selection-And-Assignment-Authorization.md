# DEC-013 — انتخاب مقیاس‌پذیر شرکت‌کنندگان و مجوز تخصیص وظیفه

| مشخصه | مقدار |
|---|---|
| وضعیت | `Agreed` |
| نسخه | `CAS UI Workspace v8` |
| دامنه | Calendar / Directory Search / Organization Scope / Task Assignment |
| صفحات مرتبط | `02_UI_UX/Employee/Calendar.md` |

## مسئله

نمایش فهرست کامل کارکنان در Modal رویداد برای سازمان بزرگ مقیاس‌پذیر نیست و مجوز دعوت به رویداد را با مجوز تخصیص وظیفه مخلوط می‌کند.

## تصمیم اصلی

انتخاب شرکت‌کنندگان در Selector مستقل روی Modal رویداد انجام می‌شود. این Selector جست‌وجومحور، فیلترپذیر و Server-side است و هیچ‌گاه همه کارکنان را یک‌جا بارگذاری نمی‌کند.

## رفتار جست‌وجو

- حالت اولیه فقط پیشنهادهای محدود مانند افراد اخیر، زیرمجموعه مستقیم و اعضای پرتکرار را نمایش می‌دهد.
- جست‌وجوی آزاد پس از حداقل دو نویسه اجرا می‌شود.
- جست‌وجو روی نام، نام خانوادگی، عنوان شغلی، واحد و شناسه پرسنلی مجاز است.
- نتایج صفحه‌بندی یا Virtualized هستند؛ حد پیشنهادی هر درخواست ۲۰ رکورد است.
- فیلترها شامل واحد، محدوده سازمانی و «زیرمجموعه‌های من» است.
- پاسخ جست‌وجو فقط افراد مجاز برای مشاهده/دعوت را برمی‌گرداند.

## انواع رابطه سازمانی

هر نتیجه باید رابطه مؤثر با کاربر جاری را مشخص کند:

- زیرمجموعه مستقیم
- زیرمجموعه غیرمستقیم مجاز
- عضو واحد خود
- همکار خارج از زنجیره مدیریتی
- شخص صرفاً قابل دعوت

## نوع ارسال برای هر فرد

برای زیرمجموعه‌ای که کاربر مجوز تخصیص وظیفه به او دارد:

- دعوت‌نامه تقویمی
- وظیفه در کارتابل
- دعوت‌نامه و وظیفه

برای فرد خارج از محدوده تخصیص:

- فقط دعوت‌نامه تقویمی

کنترل نوع ارسال فقط پس از انتخاب فرد ظاهر می‌شود. برای فرد فاقد مجوز Task، نوع ارسال Read-only است و علت محدودیت نمایش داده می‌شود.

## مدل مجوز

مجوز تخصیص وظیفه صرفاً با عنوان «مدیر» یا حضور در UI تعیین نمی‌شود. Backend باید موارد زیر را بررسی کند:

- شرکت فعال
- انتساب سازمانی معتبر در تاریخ رویداد
- رابطه مدیریتی مستقیم یا غیرمستقیم طبق سیاست سازمان
- Capability تخصیص وظیفه
- ACL و Record Rule مدل Task/Action
- محدودیت‌های تفویض اختیار

دعوت به جلسه و تخصیص وظیفه دو Operation مستقل هستند.

## تجربه انتخاب

- انتخاب‌شده‌ها در بالای Selector به‌صورت Chip نمایش داده می‌شوند.
- حذف فرد از Chip باید انتخاب را لغو کند.
- Selector دکمه‌های «تأیید شرکت‌کنندگان» و «انصراف» دارد.
- تأیید، انتخاب‌ها را به Modal اصلی برمی‌گرداند و Modal اصلی را نمی‌بندد.
- Modal اصلی خلاصه افراد انتخاب‌شده و امکان ویرایش را نشان می‌دهد.

## ممنوعیت‌ها

- بارگذاری تمام کارکنان در Client ممنوع است.
- Dropdown روش ارسال برای فرد انتخاب‌نشده نمایش داده نمی‌شود.
- یک کنترل عمومی مبهم برای روش ارسال همه افراد وجود ندارد؛ در صورت نیاز آینده، فقط به‌عنوان مقدار پیش‌فرض گروهی با Override صریح مجاز است.
- غیرفعال‌کردن گزینه Task در Frontend جایگزین کنترل سرور نیست.

## ماژول‌های متأثر

- `cas_workspace`: Modal، Selector، Layering و State
- `hr`/Employee Directory: جست‌وجوی کارکنان
- Organization/Hierarchy Service: رابطه مدیریتی
- Calendar Integration: دعوت‌نامه و Attendee
- `cas_action_hub` یا `cas_personal_task`: ایجاد وظیفه معتبر
- Notification/Mail: ارسال دعوت و اعلان

## معیار پذیرش

- در سازمان با هزاران کارمند، زمان بازشدن Modal وابسته به تعداد کل کارکنان نباشد.
- نتیجه جست‌وجو مجوزهای مؤثر را رعایت کند.
- کاربر نتواند با RPC مستقیم برای فرد غیرمجاز Task ایجاد کند.
- انتخاب‌ها پس از بستن Selector در Modal اصلی حفظ شوند.
