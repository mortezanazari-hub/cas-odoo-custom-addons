## زمینه

در نسخه ۴ و Iterationهای اولیه، اندازه برخی متن‌ها بسیار کوچک بود و Accent Color فقط روی بخش محدودی از رابط اثر می‌گذاشت. همچنین Dark Mode وجود نداشت یا به همه اجزا اعمال نمی‌شد.

## تصمیم

1. هیچ متن کاربردی کمتر از ۱۲ پیکسل نیست.
2. سه حالت خوانایی فشرده، استاندارد و بزرگ وجود دارد.
3. حالت فشرده فاصله‌ها را کم می‌کند و نباید متن را ناخوانا کند.
4. Accent Color باید Sidebar، وضعیت فعال، دکمه‌ها، Tabها، Chipها، Focus State، Widgetها، Drawer، گفتگو و تقویم را پوشش دهد.
5. Light Mode و Dark Mode باید تمام Shell و اجزای عمومی را پوشش دهند.
6. رنگ‌های وضعیت معنایی مانند خطر، هشدار و موفقیت از Accent مستقل باقی می‌مانند.
7. Theme سازمانی می‌تواند گزینه‌های مجاز کاربر را محدود کند.
8. Design Tokenها باید مرجع واحد رنگ، متن، Surface، Border، Shadow، Radius و Typography باشند.

## گزینه‌های ردشده

- تغییر Accent فقط برای متن یا دکمه اصلی
- Dark Mode جزئی برای چند صفحه
- Font Size آزاد و بدون محدودیت
- استفاده از اندازه‌های ۶ تا ۱۱ پیکسل برای Metadata
- تغییر رنگ‌های معنایی همراه Accent

## پیامدها

### مثبت

- خوانایی بهتر در محیط اداری
- یکپارچگی تمام صفحات و نقش‌ها
- امکان شخصی‌سازی کنترل‌شده
- آمادگی بهتر برای Accessibility

### منفی یا هزینه

- نیاز به Refactor CSS پراکنده به Tokenهای مرکزی
- نیاز به تست Contrast در تمام Accentها و Dark Mode
- نیاز به سیاست سازمانی Theme

## وضعیت پیاده‌سازی

تصمیم محصول و UX مورد توافق است. Design Token نهایی و ذخیره Preference در Production هنوز `Needs Review` است.
