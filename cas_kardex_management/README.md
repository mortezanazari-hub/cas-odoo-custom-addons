# CAS Kardex Management

> نسخه: `19.0.1.0.1` · Odoo 19 Community

کاردکس دقیقه‌ای بر مبنای شیفت و حضور قطعی، درخواست مرخصی/مأموریت، اضافه‌کاری و قفل دوره.

## اجزای ماژول

- `cas.kardex.day`: محاسبه روزانه دقیقه‌ای
- `cas.kardex.period`: دوره و قفل ماه
- `cas.kardex.reopen`: بازگشایی محدود
- `cas.attendance.request`: مرخصی/مأموریت
- `cas.overtime.request`: تأیید اضافه‌کاری انجام‌شده
- `cas.overtime.authorization`: مجوز خودکار تاریخ‌دار
- `cas.kardex.approval.mixin`: قواعد تأیید

## نقش‌ها

کاربر، سرپرست، مدیر و مدیرعامل کاردکس.

## روش کار

1. روز کاردکس از shift day، attendance day و درخواست‌های تأییدشده محاسبه می‌شود.
2. کارمند درخواست ساعتی/روزانه/چندروزه را ارسال و سرپرست تصمیم می‌گیرد.
3. اضافه‌کاری انجام‌شده به مدیر و سپس مدیرعامل می‌رود مگر مجوز معتبر خودکار وجود داشته باشد.
4. پس از مهلت، دوره قفل می‌شود و محاسبه/ویرایش متوقف می‌گردد.
5. بازگشایی فقط با دامنه، تاریخ، دلیل و اختیار مدیرعامل انجام می‌شود.

## منوها

کاردکس من، درخواست‌های من، کاردکس روزانه، رسیدگی درخواست/اضافه‌کاری، دوره‌ها و بازگشایی.

## نصب و ارتقا

وابستگی‌ها: `cas_attendance_core`، `cas_shift_management`، `hr`، `mail`.

```bash
./odoo-bin -d <database> -i cas_kardex_management --stop-after-init
./odoo-bin -d <database> -u cas_kardex_management --stop-after-init
```

## قواعد کلیدی

- همه مقادیر محاسباتی بر حسب دقیقه ذخیره می‌شوند.
- تعطیلات در پوشش درخواست‌ها لحاظ می‌شوند.
- دوره قفل‌شده فقط از مسیر بازگشایی ممیزی‌پذیر تغییر می‌کند.

## آزمون‌ها

آزمون‌های دقیقه‌ای، request، overtime، lock/reopen و security.
