# معماری و راهنمای استفاده از CAS Core

## هدف و مرز

هسته فنی و نقطه اتکای مشترک افزونه‌های CAS؛ عمداً فرایند کسب‌وکاری یا منوی مستقل ندارد.

این ماژول مالک همین دامنه است؛ داده ماژول‌های دیگر باید با رابطه یا API مصرف شود و موازی بازسازی نشود.

## وابستگی‌ها

- `base`

## اجزای تشکیل‌دهنده

- `__manifest__.py`: مرز وابستگی و هویت فنی
- `__init__.py`: نقطه بارگذاری بسته

## نقش‌ها و امنیت

فاقد نقش مستقل؛ مجوزهای استاندارد Odoo.

منو معیار نهایی امنیت نیست؛ ACL، record rule و کنترل سمت سرور باید دسترسی لینک مستقیم و RPC را نیز محدود کنند.

## رابط کاربری

منوی مستقل ندارد.

Workspace ممکن است مسیر ساده‌تری به همین قابلیت‌ها بدهد، اما مالکیت داده و مجوزها در این ماژول می‌ماند.

## سناریوی عملیاتی

1. پیش از افزونه‌های وابسته نصب می‌شود.
2. هیچ داده عملیاتی در آن ثبت نمی‌شود.
3. فقط قابلیت‌های واقعاً مشترک و بدون مالک دامنه در این لایه قرار می‌گیرند.

## قواعد داده و ممیزی

- نباید به ماژول دامنه‌ای وابسته شود.
- نباید محل انباشت کدهای نامرتبط شود.

در عملیات رسمی از ابطال/بایگانی و رخداد جبرانی استفاده کنید؛ حذف فیزیکی نباید سابقه تصمیم را از بین ببرد.

## نصب، ارتقا و تست

```bash
./odoo-bin -d <database> -i cas_core --stop-after-init
./odoo-bin -d <database> -u cas_core --stop-after-init
./odoo-bin -d <database> --test-enable --test-tags /cas_core -u cas_core --stop-after-init
```

سلامت با نصب ماژول‌های وابسته سنجیده می‌شود.

پس از ارتقا، نقش‌ها، record rule، یک سناریوی کامل، لاگ سرور و assetهای دسکتاپ/موبایل را بررسی کنید.

## راهنمای توسعه

- منطق دامنه را در مدل/سرویس و UI را مصرف‌کننده نگه دارید.
- XML ID و نام مدل عمومی را بدون migration تغییر ندهید.
- عملیات رسمی را ممیزی‌پذیر و تست‌های batch/RPC را پوشش دهید.
- ادغام Action Hub فقط اقدام و لینک منبع را منتشر می‌کند و مالکیت رکورد را منتقل نمی‌کند.
- هر تغییر schema یا چرخه وضعیت باید همراه migration، آزمون و اصلاح این مستند باشد.

## عیب‌یابی

- **منو دیده نمی‌شود:** نقش، implied group و وابستگی‌ها را کنترل کنید.
- **رکورد باز نمی‌شود:** record rule شرکت، مالک و مسئول را بررسی کنید.
- **تغییر وضعیت رد می‌شود:** وضعیت جاری، نقش و فیلدهای الزامی را کنترل کنید.
- **UI قدیمی است:** ماژول را ارتقا، asset را بازسازی و کش مرورگر را پاک کنید.
- **نصب شکست خورد:** اولین traceback لاگ سرور را مبنا قرار دهید.
