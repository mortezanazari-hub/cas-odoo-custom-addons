# CAS Action Hub

> نسخه: `19.0.1.1.0` · Odoo 19 Community

کارتابل یکپارچه اقدام‌های ماژول‌های مختلف با مسئول، اولویت، مهلت، SLA و لینک منبع.

## اجزای ماژول

- `cas.action.item`: آیتم و adapter منبع
- `cas.action.event`: رخداد تغییرناپذیر
- `cas.action.sla.rule`: SLA و تشدید
- سرویس sync و cron
- systray شمارنده

## نقش‌ها

کاربر کارتابل و مدیر؛ کاربران داخلی عضو نقش کاربر می‌شوند.

## روش کار

1. ماژول منبع اقدام را منتشر می‌کند یا adapter آن را کشف می‌کند.
2. Hub مسئول، اولویت، موعد و لینک منبع را نگه می‌دارد.
3. کاربر اقدام را از کارتابل باز و روی رکورد اصلی انجام می‌دهد.
4. همگام‌سازی وضعیت را از منبع تازه می‌کند.
5. Cron SLA را بررسی و رخداد تشدید ثبت می‌کند.

## منوها

کارتابل من، همه اقدام‌های قابل مشاهده، همگام‌سازی دستی و قوانین SLA.

## نصب و ارتقا

وابستگی‌ها: `mail`.

```bash
./odoo-bin -d <database> -i cas_action_hub --stop-after-init
./odoo-bin -d <database> -u cas_action_hub --stop-after-init
```

## قواعد کلیدی

- Hub مالک فرایند منبع نیست.
- دیدن آیتم از مجوز منبع فراتر نمی‌رود.
- رخدادها append-only هستند.
