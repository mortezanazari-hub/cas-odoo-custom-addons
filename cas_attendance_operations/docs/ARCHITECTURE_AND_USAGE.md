# معماری و راهنمای استفاده از CAS Attendance Operations

## هدف و مرز

عملیات ورود آفلاین Excel، نگاشت هویت خارجی و ثبت سریع گروهی نگهبانی.

این ماژول مالک همین دامنه است و داده ماژول دیگر را فقط از رابطه/API مصرف می‌کند.

## وابستگی‌ها

- `cas_kardex_management`

## اجزای تشکیل‌دهنده

- `cas.attendance.identity`: نگاشت شناسه خارجی به کارمند
- `cas.attendance.import`: پرونده ورود مرحله‌ای
- `cas.attendance.import.line`: ردیف آماده/نامشخص/تکراری/خطا
- `cas.guard.batch` و line: ثبت گروهی نگهبانی
- پارسر Excel مبتنی بر `openpyxl`

## نقش‌ها و امنیت

از نقش‌های Attendance Core استفاده می‌کند: نگهبان، سرپرست و مدیر.

ACL، record rule شرکت/کارمند/حوزه و کنترل مدل، معیار نهایی‌اند؛ پنهان‌کردن منو امنیت ایجاد نمی‌کند.

## منو و رابط

ثبت گروهی نگهبانی، ورود فایل دستگاه/نگهبانی و نگاشت شناسه‌های خارجی.

Workspace می‌تواند مسیر ساده‌تری بدهد، اما محاسبه و مجوز در مدل‌های همین ماژول باقی می‌ماند.

## سناریوی عملیاتی

1. فایل را با نوع، محل و دستگاه بارگذاری کنید.
2. «خواندن و آماده‌سازی» ردیف‌ها را بدون ثبت نهایی تحلیل می‌کند.
3. هویت‌های ناشناخته را نگاشت و ردیف‌های تکراری/خطا را بازبینی یا رد کنید.
4. فقط ردیف‌های ready را ثبت نهایی کنید.
5. برای ثبت دستی سریع، batch نگهبانی را تکمیل و یکجا تأیید کنید.

## قواعد داده و ممیزی

- parse با import نهایی یکی نیست.
- هویت ناشناخته نباید حدسی به کارمند متصل شود.
- تکراری‌ها باید idempotent تشخیص داده شوند.

اصلاح سوابق رسمی باید با ابطال، جایگزینی یا بازگشایی مجاز انجام شود؛ حذف فیزیکی سابقه عملیاتی مجاز نیست.

## نصب، ارتقا و تست

```bash
./odoo-bin -d <database> -i cas_attendance_operations --stop-after-init
./odoo-bin -d <database> -u cas_attendance_operations --stop-after-init
./odoo-bin -d <database> --test-enable --test-tags /cas_attendance_operations -u cas_attendance_operations --stop-after-init
```

آزمون‌های import staging، duplicate، identity و guard batch.

پس از ارتقا با کاربران نقش‌های مختلف، یک سناریوی کامل، جداسازی چندشرکتی، لاگ و فایل خروجی را کنترل کنید.

## راهنمای توسعه

- زمان محاسباتی را به دقیقه نگه دارید و فقط در UI/Excel قالب‌بندی کنید.
- قواعد مؤثر را تاریخ‌دار و سابقه رسمی را append-only طراحی کنید.
- عملیات import باید مرحله parse/review/commit و تشخیص تکراری داشته باشد.
- XML ID و نام مدل را بدون migration تغییر ندهید.
- هر تغییر محاسبه باید تست مرزی نیمه‌شب، تعطیل، قفل دوره و چندشرکتی داشته باشد.

## عیب‌یابی

- **نتیجه روز اشتباه است:** ابتدا shift day، سپس رخدادهای فعال و درخواست تأییدشده را بررسی کنید.
- **فایل وارد نمی‌شود:** قالب ستون‌ها، نگاشت هویت و ردیف‌های unmatched را ببینید.
- **رکورد قفل است:** وضعیت دوره و بازگشایی فعال را کنترل کنید.
- **کاربر داده‌ای نمی‌بیند:** employee-user، شرکت مجاز، نقش و record rule را بررسی کنید.
- **Excel ناسازگار است:** مجموع دقیقه‌ها را با سلول‌های duration مقایسه کنید، نه float ساعت.
