# تغییرات نسخه ۸ — Iteration 3

## بازطراحی انتخاب شرکت‌کنندگان تقویم

- حذف فهرست بلند کارکنان از Modal اصلی رویداد.
- تبدیل انتخاب شرکت‌کنندگان به Drawer مستقل و مقیاس‌پذیر.
- نمایش فقط افراد پیشنهادی/اخیر در حالت اولیه.
- جست‌وجو با حداقل دو حرف، فیلتر واحد و فیلتر محدوده سازمانی.
- محدودسازی نتایج نمایشی به ۲۰ مورد در Prototype؛ قرارداد Production باید Server-side، صفحه‌بندی‌شده یا Virtualized باشد.
- نمایش انتخاب‌ها به‌صورت Chip و امکان حذف سریع.
- نمایش نوع رابطه سازمانی برای هر فرد.
- فقط دعوت‌نامه برای افراد خارج از زنجیره مدیریتی.
- دعوت، وظیفه یا هر دو برای زیرمجموعه مجاز.
- نمایش کنترل نوع ارسال فقط بعد از انتخاب شخص.
- حذف کنترل عمومی و متناقض «نحوه ارسال برای همه».
- حفظ Header و Footer Modal اصلی و جلوگیری از اسکرول تودرتو.

## الزام Backend

مجوز ارسال وظیفه باید در سرور با ساختار سازمانی، انتساب معتبر، شرکت فعال، ACL و Record Rule بررسی شود. پنهان‌کردن گزینه در UI امنیت محسوب نمی‌شود.
