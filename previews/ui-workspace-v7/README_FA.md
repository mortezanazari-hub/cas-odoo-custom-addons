# CAS UI Workspace v7

نسخه نهایی فعلی Prototype فضای کار سازمانی CAS، بر پایه **نسخه ۴** و بازخوردهای مستقیم ارزیابی UX.

## خط مبنا و نسخه‌بندی

خط مبنای رسمی این نسخه، بسته `CAS_UI_Prototype_V4` است. تغییرات بعدی در جریان طراحی به‌صورت Iterationهای داخلی انجام شدند و نسخه‌های ۵ و ۶ به‌عنوان Release رسمی مستقل مستند نمی‌شوند.

```text
CAS UI Prototype v4  →  CAS UI Workspace v7
```

نسخه ۷ بازنویسی مستقل سامانه نیست. ساختار نقش‌محور، صفحات تخصصی، منطق دسترسی، Routeها، قراردادهای Odoo و پوشش ماژول‌های نسخه ۴ حفظ شده و تجربه عمومی Workspace روی همان پایه توسعه یافته است.

## مهم‌ترین تغییرات نسبت به نسخه ۴

- توسعه میزکار کاربر عادی از ثبت فعالیت روزانه به مرکز فرمان شخصی کامل
- اضافه‌شدن Briefing روزانه، کارهای من، نیازمند اقدام و جست‌وجوی سراسری
- بازطراحی Hero میزکار با ساختار منظم سه‌ردیفه
- افزایش حداقل اندازه متن کاربردی و اضافه‌شدن مقیاس خوانایی
- تغییر عنوان نمایشی «پیام‌ها» به «گفت‌وگوها» با حفظ Route فنی `messages`
- دسترسی سریع به گفتگو از Topbar و میزکار
- انتقال اطلاعات جانبی گفتگو به Drawer درخواستی
- Sidebar با Collapse/Expand پایدار و حفظ وضعیت کاربر
- Accent Color سراسری و Dark Mode
- ویجت‌های قابل جابه‌جایی با ذخیره ترتیب
- ارتفاع ثابت ویجت‌ها، اسکرول داخلی و ردیف‌های خالی برای حفظ نظم بصری
- تقویم تعاملی روز، هفته و ماه داخل میزکار
- نمای ماهانه شمسی با عنوان ماه، سال، روزهای هفته و ناوبری ماه قبل/بعد
- ادغام میانبرهای تکراری در نوار عملیات میزکار
- حفظ صفحات تخصصی نگهبانی، دبیرخانه، حضور، شیفت، فرم، گردش‌کار، اسناد و مدیریت سامانه

## اصول معماری حفظ‌شده

- هیچ تغییری در Odoo Core انجام نمی‌شود.
- Workspace مالک داده کسب‌وکاری نیست و فقط پوسته، ناوبری، Widget و Action Layer را ارائه می‌کند.
- داده‌ها باید از مدل‌های واقعی Odoo با رعایت ACL و Record Rule خوانده شوند.
- استفاده از `sudo` برای دورزدن کنترل دسترسی مجاز نیست.
- Jalali فقط لایه ورود و نمایش تاریخ است و مقدار استاندارد ذخیره‌شده Odoo را تغییر نمی‌دهد.
- رابط باید کاملاً فارسی، راست‌چین، واکنش‌گرا و مستقل از ظاهر پیش‌فرض Odoo باقی بماند.

## وضعیت Prototype

- داده‌ها نمایشی هستند.
- تعامل‌ها برای ارزیابی UX پیاده‌سازی شده‌اند.
- اتصال نهایی به مدل‌ها، Mail/Bus، Calendar و سرویس‌های Odoo در مرحله پیاده‌سازی انجام می‌شود.
- این مسیر در حال حاضر شامل مستندات نسخه است؛ فایل‌های اجرایی تنها پس از تأیید جداگانه ثبت می‌شوند.
