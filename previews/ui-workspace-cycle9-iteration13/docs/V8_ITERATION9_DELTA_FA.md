# نسخه ۸ — Iteration 9: قرارداد اسکرول صفحه گفت‌وگو

## مسئله
صفحه گفت‌وگو نباید به‌صورت یک Route کامل اسکرول شود؛ با این حال کاربر باید بتواند در هر دو ناحیه محتوایی، علاوه بر Wheel و Scrollbar، از Auto-scroll بومی مرورگر با دکمه وسط موس استفاده کند.

## تصمیم نهایی
- Viewport خود صفحه گفت‌وگو ثابت است.
- Header گفتگو و Composer ثابت می‌مانند.
- فقط `conversation-list` و `chat-body` اسکرول عمودی مستقل دارند.
- هر دو ناحیه از `overflow-y:auto` بومی مرورگر استفاده می‌کنند و رویداد دکمه وسط موس در آن‌ها مسدود نمی‌شود.
- Scroll تودرتوی صفحه و انتقال اسکرول به Route اصلی حذف می‌شود.
- در نمایش باریک نیز ارتفاع صفحه ثابت است و فضای موجود میان فهرست گفتگو و پنل چت تقسیم می‌شود.

## معیار پذیرش
1. کلیک دکمه وسط داخل فهرست گفتگو Auto-scroll همان فهرست را فعال کند.
2. کلیک دکمه وسط داخل پیام‌ها Auto-scroll پیام‌ها را فعال کند.
3. کلیک دکمه وسط روی Header یا Composer باعث اسکرول کل Route نشود.
4. Composer در پایین پنل گفتگو باقی بماند.
5. Wheel، نوار اسکرول، Keyboard scrolling و Touch scrolling همچنان کار کنند.
6. سایر Routeها اسکرول اصلی بومی خود را حفظ کنند.
