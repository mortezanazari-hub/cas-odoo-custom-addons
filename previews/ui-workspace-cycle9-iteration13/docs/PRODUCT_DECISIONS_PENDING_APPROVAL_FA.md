# تصمیمات طراحی در انتظار تأیید

این سند برای قرارگیری همراه Prototype در GitHub آماده شده، اما وضعیت آن هنوز «پیشنهادی» است.

1. Workspace به‌عنوان مرکز فرمان شخصی، نه داشبورد صرف.
2. تفکیک «کارهای من»، «تقویم» و «گزارش فعالیت».
3. عدم استفاده از Project و To-do استاندارد Odoo برای کارهای شخصی CAS.
4. عدم استفاده از Kanban در مدیریت کارهای شخصی.
5. صفحه مستقل تقویم با سه نمای روز، هفته و ماه.
6. استفاده احتمالی از Backend تقویم Odoo با رابط کاملاً اختصاصی.
7. حفظ زیرساخت Mail/Discuss و طراحی پوسته اختصاصی پیام‌رسانی CAS.
8. Action-driven و Search-driven بودن تجربه کاربری.
9. ثبت قابلیت‌های هر ماژول تخصصی در Command Layer بدون انتقال منطق تخصصی به Workspace.
10. عدم تغییر هسته Odoo.

پس از ارزیابی v6، وضعیت هر بند باید به Approved، Revised یا Rejected تغییر کند.
