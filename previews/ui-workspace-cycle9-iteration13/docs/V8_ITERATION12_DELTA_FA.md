# نسخه ۸ — Iteration 12

## دامنه
تبدیل صفحه ثابت «ثبت گزارش روزانه» به پوسته پویا مبتنی بر `cas_work_report` و هسته فرم‌ساز.

## تغییرات نمایشی
- عنوان و ساختار فرم براساس نقش/واحد/انتساب تغییر می‌کند.
- اطلاعات هویتی، واحد، شیفت و حضور به‌صورت خودکار و فقط‌خواندنی نمایش داده می‌شوند.
- بخش فعالیت‌ها از فرم منتشرشده و پروفایل گزارش کار ساخته می‌شود.
- نمونه‌های تخصصی برای نگهبانی، دبیرخانه، فناوری اطلاعات، مدیریت واحد، مدیریت عالی و عملیات اضافه شد.
- پنل کنترل گزارش اکنون پروفایل فعال، مهلت، مسیر تأیید و نسخه Snapshot فرم را نمایش می‌دهد.
- پیام «چرا این فرم نمایش داده شد؟» منطق Resolve پروفایل را برای کاربر شفاف می‌کند.

## تصمیم معماری
- `cas_form_core` مالک Schema، Version، Validation و Submission است.
- `cas_work_report` مالک Lifecycle، Profile Resolution، Deadline، Approval، Attendance Reconciliation و Reporting است.
- `cas_workspace` مالک Route و تجربه نمایشی است.
- برای هر فرم گزارش کار ماژول جدا ساخته نمی‌شود.
- هر گزارش به نسخه فرم و Snapshot پایدار متصل می‌شود.
