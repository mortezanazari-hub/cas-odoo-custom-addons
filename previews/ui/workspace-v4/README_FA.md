# نمونه جامع رابط کاربری CAS

این بسته یک **نمونه تعاملی Front-end** برای فضای کار سازمانی CAS است که بر اساس مستند جامع ۲۴ ماژول تهیه شده است. طراحی کاملاً فارسی، راست‌چین، واکنش‌گرا و مستقل از ظاهر پیش‌فرض Odoo است.

## اجرای سریع

### روش ۱: اجرای مستقیم
فایل `index.html` را در مرورگر باز کنید.

### روش ۲: وب‌سرور محلی (پیشنهادی)

ویندوز:

```bat
preview.bat
```

Linux/macOS:

```bash
./preview.sh
```

سپس آدرس زیر را باز کنید:

```text
http://localhost:8080
```

## قابلیت‌های نمونه

- انتخاب چند کاربر آزمایشی با ترکیب متفاوت نقش‌ها و نمایش پویای منو بر اساس دسترسی مؤثر؛
- نمایش منو و صفحه آغاز متناسب با نقش؛
- بیش از ۴۰ route تخصصی برای فرم، گردش‌کار، تأیید، کارتابل، مکاتبات، اسناد، دبیرخانه، شیفت، حضور، نگهبانی، کاردکس و گزارش کار؛
- طراح دیداری فرم و گردش‌کار؛
- ثبت نگهبانی کارت‌محور با جست‌وجوی فرد، فیلتر واحد، تغییر دستی ساعت و نمایش لاگ آخرین رخدادها؛
- صفحات مدیریتی و مدیرعامل با KPI، ریسک و روند؛
- مرکز مدیریت CAS شامل کاربران، نقش‌ها، ساختار سازمانی، سمت‌ها، انتساب‌ها، تنظیمات دامنه‌ای، گزارش دسترسی و ممیزی؛
- صفحات فنی برای سلامت سامانه، تاریخ جلالی، Storage/OCR و تنظیمات Workspace؛
- طراحی Responsive برای دسکتاپ، تبلت و موبایل؛
- بدون کتابخانه یا CDN خارجی.

## نکته مهم درباره پیاده‌سازی Odoo

داده‌های این نمونه نمایشی‌اند. در نسخه Odoo باید داده‌ها با RPC از مدل‌های واقعی خوانده شوند. Workspace نباید مالک داده کسب‌وکاری شود و نباید برای دورزدن ACL یا Record Rule از `sudo` استفاده کند. قرارداد پیشنهادی اتصال در فایل `docs/ODOO_INTEGRATION_CONTRACT_FA.md` آمده است.

## مدل دسترسی نسخه جدید

در این نسخه، انتخاب‌گر بالای صفحه «کاربر آزمایشی» است، نه تغییر نقش آزاد. هر کاربر آزمایشی مجموعه‌ای از نقش‌های امنیتی و فضاهای کاری دارد و منو با Access Resolver نمایشی از روی همان داده‌ها ساخته می‌شود. در نسخه Production باید کاربر از Session واقعی Odoo دریافت شود و انتخاب‌گر آزمایشی حذف گردد. پنهان‌شدن منو جایگزین ACL، Record Rule و کنترل سمت سرور نیست.

## بازطراحی منوی تجربه‌محور در نسخه ۳

در نسخه ۳، منوی روزمره از ساختار فنی ماژول‌ها جدا شده است. کاربر عادی فقط «خانه»، «کارهای من»، «ثبت درخواست»، «پیگیری درخواست‌ها»، «گزارش روزانه»، «حضور و شیفت» و در صورت داشتن مجوز «مکاتبات» را می‌بیند. گزینه‌های فرم‌های قابل ثبت، ثبت‌های من، گردش‌کارهای من، کاردکس، اضافه‌کاری و جابه‌جایی شیفت حذف نشده‌اند؛ بلکه داخل صفحات یکپارچه قرار گرفته‌اند.

منو با capabilityهای مؤثر کاربر ساخته می‌شود. نقش‌های امنیتی فقط یکی از منابع تولید capability هستند و این ساختار برای اتصال به گروه‌های Odoo، ACL، Record Rule، سمت، واحد، جانشینی و دسترسی موقت آماده شده است.

فضاهای تخصصی جدید:

- مرکز سرپرستی و صفحه «تیم و درخواست‌ها»
- فضای «ثبت و پایش تردد» برای نگهبانی
- مرکز مدیریت CAS برای مدیر سامانه
- ورودی یکپارچه ثبت و پیگیری درخواست‌ها
- صفحه ترکیبی حضور، شیفت، کارکرد و اصلاحات
